O co-coach em uma frase

É um sistema pessoal de aprendizado que transforma links e repositórios do GitHub em cards de conhecimento acessíveis pelo celular — e distribui habilidades (skills) para outros projetos automaticamente.

📥 Você envia um link → 🤖 IA sumariza → 📱 Card no feed → 🛠️ Skills distribuídas

As 3 camadas do sistema

Cada camada tem uma função diferente. Juntas, elas formam o co-coach.

🗂️

Camada 1 — Knowledge Base

kb/

O "cérebro" do sistema. Armazena tudo que você aprendeu sobre Claude Code, IA e ferramentas em arquivos de texto estruturados.

Como funciona na prática ▾

1. Você compartilha uma URL pelo iPhone (ex: um artigo ou vídeo do YouTube)

2. O iPhone cria automaticamente uma tarefa no GitHub

3. O GitHub avisa o sistema para processar o link

4. O Claude (modelo Haiku, mais rápido e barato) lê o conteúdo e extrai: título, tema, 3-5 pontos principais e uma nota de importância

5. Tudo é salvo em um arquivo kb/2026-06-25-nome-do-artigo.md

📱

Camada 2 — Feed Mobile

docs/index.html

A "vitrine" do sistema. Um site que você acessa pelo celular com todos os cards de conhecimento, filtráveis por tema.

Como funciona na prática ▾

1. Um script Python (build-site.py) lê todos os arquivos da Knowledge Base

2. Gera um site HTML estático com todos os cards

3. O GitHub publica esse site automaticamente em victorauad.github.io/co-coach

4. Sempre que você adiciona novo conteúdo, o site é reconstruído em ~2 minutos

5. Uma vez por semana (toda segunda às 8h) o site é reconstruído mesmo sem novos conteúdos

🛠️

Camada 3 — Hub de Skills

skills/

O "distribuidor" do sistema. Cria habilidades especializadas para o Claude e as distribui automaticamente para todos os seus projetos.

Como funciona na prática ▾

1. Uma skill é um arquivo de texto (SKILL.md) que ensina o Claude a fazer algo específico

2. Ex: co-coach-bigquery ensina o Claude a escrever queries otimizadas para o BigQuery

3. Quando você cria ou atualiza uma skill no co-coach, o GitHub distribui automaticamente para: iracing_analysis e alamtoco

4. Em qualquer desses projetos, você digita /co-coach-bigquery e o Claude já sabe o que fazer

Do link ao card: passo a passo

O que acontece nos bastidores quando você compartilha uma URL pelo iPhone.

📱
Passo 1 0 segundos

Você vê um artigo interessante no iPhone e usa o botão "Compartilhar → co-coach" (iOS Shortcut configurado).

📋
Passo 2 ~2 segundos

O iPhone cria automaticamente uma tarefa (Issue) no GitHub com a URL e uma etiqueta add-link.

⚙️
Passo 3 ~10 segundos

O GitHub detecta a etiqueta e aciona um robô (GitHub Actions) que baixa o conteúdo da URL.

🤖
Passo 4 ~15 segundos

O Claude Haiku (modelo de IA mais rápido e econômico) lê o conteúdo e extrai automaticamente: título, categoria, 3–5 pontos-chave e nota de relevância (1–5).

✅
Passo 5 ~2 minutos

O site é reconstruído e o card aparece em victorauad.github.io/co-coach. A tarefa no GitHub é fechada automaticamente com um ✅.

Glossário — termos técnicos explicados

Todos os termos que aparecem neste sistema, em linguagem simples.

GitHub

Site onde o código e os arquivos do co-coach ficam armazenados na nuvem. Funciona como um "Google Drive para código".

GitHub Actions

Robôs automáticos que o GitHub executa quando algo acontece (ex: um arquivo é salvo). São as automações do co-coach.

GitHub Pages

Serviço gratuito do GitHub que publica o feed mobile (victorauad.github.io/co-coach) sem precisar de servidor próprio.

Claude Haiku

Versão mais rápida e econômica do Claude, usada para sumarizar conteúdo em lote. O Claude maior (Sonnet) fica para conversas.

Skill

Um arquivo de instruções que ensina o Claude a fazer algo específico. Como um "manual de procedimento" que o Claude lê antes de responder.

Frontmatter YAML

Cabeçalho estruturado no início de cada arquivo da KB com campos como "titulo:", "tema:", "importancia:". É o que permite filtrar os cards.

iOS Shortcut

Automação no iPhone (app Atalhos) que, ao receber uma URL compartilhada, cria a tarefa no GitHub. É a "boca de entrada" do sistema.

MCP (plugins para o Claude)

Protocolo que permite ao Claude se conectar a ferramentas externas (como Figma, Google Drive, Gmail) durante uma conversa.
