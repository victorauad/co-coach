## O princípio que vale para todos os fluxos

Todos seguem o mesmo loop, que vale a pena internalizar:

1. **Contexto** — o Claude precisa enxergar os dados/arquivos (pasta aberta, CLAUDE.md, ou conector MCP).
2. **Plano** — `Shift+Tab` (Plan Mode) pra ele propor antes de executar. Você revisa.
3. **Execução supervisionada** — ele faz, você aprova cada passo sensível.
4. **Verificação** — você confere o resultado (não confie cego: rode, abra, valide).
5. **Captura** — se você vai repetir isso, vira uma *skill* ou um *comando* (veja `03-metodologias/`).

O item 5 é o que diferencia quem usa a ferramenta de quem **alavanca** a ferramenta. Toda tarefa que você fizer mais de uma vez deveria virar um atalho.
