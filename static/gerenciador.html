Configurar acesso ao GitHub

O gerenciador usa a GitHub API para ler e salvar arquivos diretamente no repositório. Configure uma vez — fica salvo no browser.

Crie em GitHub → Settings → Developer settings → Personal access tokens → Fine-grained. Permissão necessária: Contents: Read & Write nos repos co-coach, iracing_analysis, alamtoco e SMPL.

Sync de Repos

Quais repositórios recebem as skills automaticamente.

config/sync-targets.yml

Formato: um repositório por linha, ex: victorauad/co-coach. O sync distribui as skills para todos os repos listados.

Como funciona o sync

1

Você cria ou edita uma skill em skills/co-coach-*/SKILL.md

2

Quando você salva, o GitHub distribui a skill automaticamente para os outros projetos

3

O processo lê config/sync-targets.yml e envia a skill para cada repo listado

✓

A skill fica disponível em .claude/skills/ de cada repo — sem precisar copiar manualmente
